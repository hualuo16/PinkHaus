# PinkHaus — 好物测评社区

> **产品定位**：一个专注于化妆品和护肤品的真实测评内容社区。
> **产品形态**：移动端 Web App（React + Vite 原型）
> **我的角色**：产品经理

---

## 📋 项目简介

PinkHaus 的诞生源于一个观察：消费者在购买护肤品前越来越理性，她们想看到真实评价，但现有的平台要么广告泛滥、要么没有肤质参考维度、要么信息过于碎片化。

PinkHaus 的答案很简单——**以肤质为锚点、以真实评价为核心**。用户只需写几句话分享使用体验，系统自动聚合为商品评分分布，其他人搜索商品时一眼就能看到大众对它的真实评价。

##  核心功能

| 模块 | 功能 |
|------|------|
| 评价系统 | 💖 PINK（超级赞）/ ❤️ 红榜 / 💛 黄榜 / 🖤 黑榜 四档评分 |
| 商品搜索 | 关键字模糊匹配 + 语音搜索 + 品类图标浏览 |
| 社区 Feed | 所有评价按时间倒序展示，每条标注用户肤质 |
| 商品详情 | 评分分布柱状图 + 同肤质用户评价列表 |
| 用户体系 | 多步注册 → 肤质测试 → 个人主页（小红书风格） |
| 收藏系统 | 收藏商品 + 收藏他人评价 |

##  目标用户

- 关注产品功效、想了解真实口碑再下单的美妆护肤消费者
- 关注肤质匹配度、寻找同肤质用户反馈的护肤成分党
- 喜欢分享和浏览美妆护肤内容的种草/拔草人群

##  竞品差异

| vs 小红书 | 专注美妆护肤 + 强制标注肤质 + 纯评价无广告 |
|-----------|-------------------------------------------|
| vs 美丽修行 | 侧重用户评价社区而非成分数据 |
| vs 电商评价 | 独立社区 + 肤质标签 + 评分分布可视化 |
| vs 抖音快手 | 结构化评价 + 可搜索 + 可聚合 |

##  技术栈

React 18 · Vite 5 · React Router v6 · localStorage · Web Speech API
